源码介绍


系统功能：

学生端功能：

实时查看电力和热水消耗情况。

设定每日或每月节能目标，系统实时反馈节能目标的完成情况。

系统通过图表展示消耗趋势，帮助学生直观了解用电情况。

管理端功能：

查看全校宿舍的电力消耗情况。

通过数据可视化工具（如图表）展示用电数据的变化趋势。

提供用电分析，帮助管理者作出节能决策。

实时数据传输：

使用 MQTT 协议实现数据的实时传输，保证电力和热水消耗数据的准确性与及时性。

节能提醒与反馈：

系统根据学生的用电习惯与目标提供节能建议，并在消耗接近设定上限时发送提醒，帮助学生养成节约能源的习惯。

技术实现：

前端技术：

学生端使用 React Native 开发，便于在移动设备上查看和设置节能目标。

管理端使用 React.js，通过图表和表格展示宿舍楼层的整体电力消耗。

所有前端界面都通过简洁、直观的设计，确保用户体验友好。

后端技术：

Flask 框架实现后端数据服务，处理来自前端的请求，并与数据库交互。

使用 SQLite 存储学生和宿舍用电数据。

数据实时传输使用 MQTT 协议，确保数据的及时传递。

数据分析：

根据用户的用电历史，系统会对数据进行分析，发现高峰期和节能潜力，为学生和管理端提供节能建议。

代码结构：

学生端：

App.js：主要功能是展示实时的电力消耗和热水消耗，以及设置节能目标。

index.html：学生端首页，展示各项实时数据。

style.css：负责页面的样式布局，使页面美观且易于使用。

script.js：控制前端交互，如更新数据、设置节能目标等。

管理端：

App.js：管理端的主组件，用于显示电力消耗的趋势图表，并支持数据的实时更新。

index.html：管理端首页，展示全校的总电力消耗与宿舍用电数据。

style.css：管理端样式设置，确保界面清晰、整洁。

script.js：控制图表数据的更新，以及后台 API 的调用。

后端：

server.py：Flask 后端框架，提供 API 服务，处理前端请求，返回数据。

apiRoutes.py：定义了系统的接口路由，处理能耗数据请求。

database.py：负责数据库的创建与操作，存储电力消耗、热水使用记录等。

mqtt.py：负责实时数据传输，使用 MQTT 协议确保数据的及时更新。

dataBackup.py：定期备份数据库，防止数据丢失。

energyDataModel.py：插入能耗数据到数据库，提供接口支持。

数据库：

energy_management.db：存储学生的用电数据、热水消耗数据和节能目标等信息。

设计亮点与创新：

实时数据展示与节能目标管理：学生可以直观地看到自己的能耗情况，并通过设定节能目标来控制消耗。

节能提醒与推送：系统根据数据变化，及时提醒学生是否超出设定的用电范围，帮助其调整使用习惯。

管理端数据分析与可视化：管理端通过图表展示用电数据，帮助学校做出合理的节能决策。
